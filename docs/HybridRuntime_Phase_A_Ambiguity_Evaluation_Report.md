# HybridRuntime Research Phase A
## Ambiguity Evaluation Report

**実行日:** 2026-06-11  
**対象:** Phase A-1 / Phase A-2  
**実行コマンド:** `cd Test && cargo test --test hybrid_runtime_phase_a -- --nocapture`  
**結果:** PASS (3 passed, 0 failed)

## 1. 結論

曖昧度は単一要因では説明できない。今回の観測では、少なくとも次の変数を分離する必要がある。

1. 候補分布の不確実性: Shannon entropy
2. 候補数の実効的な大きさ: `2^entropy`
3. 上位候補の競合: 1位と2位の確率差
4. 候補間の意味的近接性: 平均 cosine similarity
5. 証拠間の対立: 証拠が作る候補順位の不一致
6. 未対応情報: 意味モデルに存在しない証拠の比率

したがって、`曖昧度 = 候補数`、`曖昧度 = 意味距離`、`曖昧度 = entropy`
のいずれか一つだけを採用する仮説は支持されない。

## 2. 実験方法

候補を7属性の実数ベクトルで表現した。

`Animal, Canine, Domestic, Wild, Pet, Vehicle, Planet`

入力証拠に対する候補尤度を積で合成し、正規化した候補確率から次を観測した。

- `H`: Shannon entropy (bits)
- `Hn`: 候補数で正規化したentropy
- `Neff`: 実効候補数 `2^H`
- `P1`: 最上位候補確率
- `Margin`: 1位確率 - 2位確率
- `Density`: 候補間平均cosine similarity
- `Conflict`: 証拠が形成する候補順位の不一致
- `Unsupported`: 未対応証拠率

これは評価式の正式仕様ではなく、要因を独立観測するための研究モデルである。

## 3. Phase A-1 結果

| ID | H | Hn | Neff | P1 | Margin | Density | 判定 |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | --- |
| AE-01 | 0.000 | 0.000 | 1.000 | 1.000 | 1.000 | 0.000 | 支持 |
| AE-02 | 1.000 | 1.000 | 2.000 | 0.508 | 0.015 | 0.674 | 支持 |
| AE-03 | 2.322 | 1.000 | 5.000 | 0.200 | 0.000 | 1.000 | 支持 |
| AE-04 | 0.579 | 0.365 | 1.494 | 0.895 | 0.823 | 0.759 | 部分支持 |
| AE-05 | 0.160 | 0.101 | 1.117 | 0.980 | 0.970 | 0.024 | 支持 |

### AE-01 Single Candidate

候補が一つの場合、`H=0`、`Neff=1` となり下限を確認した。

### AE-02 / AE-03 Candidate Count

生entropyは `0.000 -> 1.000 -> 2.322` と増加した。候補が等確率なら、
候補数は曖昧度の絶対量を増加させる。

ただし正規化entropyはAE-02とAE-03でともにほぼ1.0だった。つまり、
「候補数による絶対不確実性」と「分布の均等さ」は別の量である。

### AE-04 Near Candidates

候補密度は `0.759` と高かったが、`Domestic` がDogを強く識別し、
`P(Dog)=0.895`、`H=0.579` まで低下した。

よって「候補が意味的に近いほど、必ず曖昧度が高い」という期待は支持されない。
近接性は曖昧度そのものではなく、証拠が不足または弱い場合に競合を持続させる要因である。

### AE-05 Far Candidates

候補密度は `0.024` と低く、Animal証拠だけでDogが `0.980` まで優勢になった。
AE-04よりentropy、実効候補数、競合がすべて低く、意味的に遠い候補は
関連証拠によって早期に排除されることを確認した。

## 4. Phase A-2 結果

| 条件 | H | Hn | Neff | P1 | Margin | Conflict | Unsupported |
| --- | ---: | ---: | ---: | ---: | ---: | ---: | ---: |
| Animal | 1.585 | 1.000 | 3.000 | 0.333 | 0.000 | 0.000 | 0.000 |
| + Canine | 1.547 | 0.976 | 2.923 | 0.391 | 0.012 | 0.500 | 0.000 |
| + Domestic | 0.579 | 0.365 | 1.494 | 0.895 | 0.823 | 0.405 | 0.000 |
| + Pet | 0.055 | 0.035 | 1.039 | 0.994 | 0.990 | 0.322 | 0.000 |
| + Wild | 1.302 | 0.821 | 2.466 | 0.616 | 0.351 | 0.580 | 0.000 |
| + RandomNoise | 0.579 | 0.365 | 1.494 | 0.895 | 0.823 | 0.405 | 0.250 |

### AD-01 Evidence Increase

識別力のある証拠を追加すると、entropyは
`1.585 -> 1.547 -> 0.579 -> 0.055` と単調減少した。

ただし情報量の増加だけでは不十分である。`Canine` はDogとWolfをほぼ同程度に支持し、
減少量は小さかった。曖昧度を下げるのは証拠数ではなく、候補を識別する情報利得である。

### AD-02 Contradiction

`Domestic` までの整合条件と比較して `Wild` を加えると、
entropyは `0.579 -> 1.302`、実効候補数は `1.494 -> 2.466`、
Conflictは `0.405 -> 0.580` に増加した。

矛盾は候補分布を再拡散させるため、entropyと証拠対立を同時に観測する必要がある。

### AD-03 Noise Injection

未知の `RandomNoise` を加えても候補分布は変化せず、`Unsupported=0.25` のみ増加した。
これは「未知情報を尤度更新に使わず隔離する」モデルではノイズ耐性が成立することを示す。

一方、未対応証拠率を記録しなければ、モデルが情報を無視した事実を監査できない。
ノイズ耐性と情報欠落を区別するため、`Unsupported` は必須である。

## 5. 仮説評価

| 仮説 | 評価 | 根拠 |
| --- | --- | --- |
| 候補数に依存する | 条件付き支持 | 等確率時の生entropyは候補数とともに増加 |
| 意味距離に依存する | 条件付き支持 | 遠い候補は排除しやすいが、近さだけでは高曖昧にならない |
| 競合度に依存する | 強く支持 | 上位marginとentropyが各ケースを安定して区別 |
| 情報量に依存する | 修正して支持 | 証拠数ではなく情報利得・識別力に依存 |
| 矛盾に依存する | 支持 | entropyとConflictが同時増加 |
| ノイズで必ず増加する | 不支持 | 未知証拠を隔離すれば分布は不変 |

## 6. AmbiguityEvaluator v1への要件候補

正式仕様策定時には、少なくとも次の出力を持つべきである。

```text
candidate_entropy
normalized_entropy
effective_candidate_count
top_candidate_probability
top_two_margin
semantic_candidate_density
evidence_conflict
unsupported_evidence_ratio
```

単一の `ambiguity_score` が必要な場合も、上記の観測値を消去せず、
合成根拠と重みをtraceに残す必要がある。

## 7. 制約

- 属性値は制御実験用の手動モデルであり、学習済みembeddingではない。
- 事前確率は一様とした。
- 証拠間の条件付き独立を仮定した。
