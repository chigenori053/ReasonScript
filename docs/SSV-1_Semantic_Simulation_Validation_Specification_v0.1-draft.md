# SSV-1 Semantic Simulation Validation Specification

## Document Information

- Specification: SSV-1 Semantic Simulation Validation
- Version: 0.1-draft
- Target: ReasonScript Semantic Language v0.2
- Status: Experimental Validation
- Runtime mapping: `RuntimeReal`
- Prerequisites:
  - SemanticUnit
  - SemanticRelation
  - SCV-1
  - Reasoning Space
  - SemanticPlan

## Purpose

SSV-1 validates that a `SemanticPlan` can be evaluated against a validated
`ReasoningSpace` to produce a structured and reproducible semantic trajectory.

Semantic Simulation does not establish truth, execute external effects, store
knowledge, or mutate the Reasoning Space. It evaluates whether and how a goal
can be reached under the current graph and plan constraints.

## Formal Model

```text
Reasoning Space
       |
SemanticPlan
       |
SemanticSimulation
       |
SimulationResult
```

No knowledge object is generated by SSV-1.

## SemanticPlan

The plan contains:

- `start`
- `goal`
- optional `avoid_nodes`
- optional `max_distance`

Constraints are evaluated while searching for a trajectory. A constrained plan
that cannot reach its goal produces an unsuccessful `SimulationResult`.

## Operations

### simulate

```rust
simulation.simulate(&space, &plan)
```

Validates the space and supplied plan, evaluates the shortest allowed
trajectory, and returns a `SimulationResult`.

### simulate_goal

```rust
simulation.simulate_goal(&space, start, goal)
```

Constructs an unconstrained plan and evaluates it.

### simulate_goal_with_constraints

```rust
simulation.simulate_goal_with_constraints(
    &space,
    start,
    goal,
    constraints,
)
```

Constructs and evaluates a constrained plan.

### predict

```rust
simulation.predict(&space, current)
```

Returns all directed semantic states reachable from the current state in
deterministic breadth-first order.

### simulate_graph

`simulate_graph` is a validation-boundary helper. It attempts to promote a raw
`ReasonGraph` to a `ReasoningSpace` before simulation. This operation exists to
verify that invalid graphs are rejected by SSV-1 and is not a replacement for
the canonical `ReasoningSpace` input.

## SimulationResult

```text
SimulationResult
├─ success
├─ path
├─ distance
├─ cost
├─ confidence
├─ trace
└─ predicted_states
```

### Successful result

- `path` contains start through goal.
- `distance = path.len() - 1`.
- `cost` is the sum of edge costs.
- `confidence` is the product of edge confidences.
- `trace.states` equals `path`.
- `trace.steps.len() = distance`.
- `predicted_states` contains the future states on the selected trajectory.

### Unsuccessful result

An existing but unreachable goal is not a simulation-system error:

- `success = false`
- `path = [start]`
- `distance = 0`
- `cost = 0`
- `confidence = 0`
- `trace.states = [start]`
- `trace.steps = []`
- `predicted_states` contains states that remain reachable from `start`

Missing nodes, invalid graph structure, invalid costs, and invalid confidence
values are typed simulation errors.

## Edge Metrics

`Edge` contains independent metrics:

- `cost`: finite value greater than or equal to zero
- `confidence`: finite value in `[0.0, 1.0]`

The default edge cost is `1.0`; the default confidence is `1.0`.
Deserializing older graph data without `confidence` uses the default.

Closure-generated relations compose:

```text
derived cost       = first cost + second cost
derived confidence = first confidence * second confidence
```

## Validation Rules

1. The input is a valid `ReasoningSpace`.
2. The graph satisfies SCV-1.
3. Start and goal nodes exist.
4. Every consecutive path state has a direct graph edge.
5. Every path edge therefore satisfies SCV-1.
6. Every edge cost is finite and non-negative.
7. Every edge confidence is finite and within `[0.0, 1.0]`.
8. Repeated simulation on the same graph and plan returns the same result.
9. Every path state is recorded in `trace.states`.
10. Every path transition is recorded in `trace.steps`.
11. Simulation does not mutate the Reasoning Space.

## Non-Goals

- Truth validation
- Real-world correctness
- Temporal causality
- Spatial causality
- External execution
- Persistent storage
- Knowledge generation

## Adoption Criteria

SSV-1 can be adopted when:

1. SSV-001 through SSV-007 pass.
2. Simulation remains deterministic and serializable.
3. SCV-1 remains enforced at the simulation boundary.
4. Reasoning Space remains unchanged by simulation.
5. Cost, confidence, trace, and predicted states are reproducible.
